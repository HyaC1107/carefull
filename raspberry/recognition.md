# Carefull 얼굴 인식 시스템 (High Precision Recognition)

본 문서는 Carefull 라즈베리파이 디바이스에 적용된 고도화된 얼굴 인식 및 인증 로직을 설명합니다.

## 1. 핵심 개선 사항

### A. 색상 채널 최적화 (BGR to RGB)
- **이유**: 대부분의 딥러닝 얼굴 인식 모델(MobileFaceNet 등)은 RGB 채널 순서로 학습됩니다. OpenCV의 기본 출력인 BGR을 그대로 사용할 경우 인식률이 급격히 저하됩니다.
- **적용**: 모델 입력 직전 `cv2.cvtColor(img, cv2.COLOR_BGR2RGB)`를 수행하여 정확도를 극대화했습니다.

### B. 정사각형 고정 배율 크롭 (Square Cropping)
- **이유**: 카메라와의 거리에 따라 얼굴 박스의 크기와 비율이 변하면 임베딩 값에 편차가 발생합니다.
- **적용**: 감지된 얼굴 중심점을 기준으로 얼굴 크기의 **1.45배** 정사각형 영역을 고정적으로 추출합니다. 이를 통해 거리에 상관없이 모델이 일관된 스케일의 데이터를 처리하게 되어 거리 편차를 최소화했습니다.

### C. 멀티 템플릿 매칭 (Multi-Template Matching)
- **이유**: 단일 임베딩 벡터(평균값 등)는 사용자의 미세한 표정 변화나 조명 차이를 모두 담기에 한계가 있습니다.
- **적용**: 
    1. **등록 시**: 수집된 20장 중 고품질 상위 10장의 임베딩을 리스트 형태로 저장합니다.
    2. **인증 시**: 입력 프레임을 저장된 10개의 벡터 각각과 비교하여 **최대 유사도(Best-Match)**를 채택합니다.
    3. **장점**: 본인 거부율(FRR)을 획기적으로 낮추면서도 보안 성능을 유지합니다.

## 2. 알고리즘 워크플로우

### [등록 단계]
1. 5개 방향(정면, 상, 하, 좌, 우) 안내에 따라 20개 프레임 수집.
2. 각 프레임에서 1.45x 정사각형 ROI 추출.
3. 이미지 품질(해상도) 순으로 정렬하여 상위 10개 선별.
4. 10개의 128차원 임베딩 벡터를 JSON DB에 저장.

### [인증 단계]
1. 최대 15장의 프레임을 연속 캡처.
2. 각 프레임을 저장된 10개의 템플릿과 개별 비교 (Best-Match 방식).
3. `임계값(0.85)`을 넘는 프레임이 전체의 `30%` 이상일 경우 최종 승인.

## 3. DB 구성 변경 사항 안내

현재 멀티 템플릿 방식을 지원하기 위해 로컬 DB 구조가 다음과 같이 변경되었습니다.

- **기존**: `"username": [0.1, 0.2, ...]` (단일 벡터 리스트)
- **변경**: `"username": [[0.1, ...], [0.3, ...], ...]` (벡터들의 중첩 리스트)

**서버 DB 호환성 정보**:
- 서버(MySQL)의 경우, 현재 `face_vector` 컬럼이 단일 벡터(문자열/Blob)를 저장하는 구조라면, 상위 10개의 **평균값(Mean Embedding)**을 대표로 전송하여 호환성을 유지하고 있습니다.
- 만약 서버에서도 멀티 매칭을 수행하려면 테이블 구조를 1:N(User:Templates) 관계로 확장해야 합니다. (현재는 기기 로컬 인증 성능 강화에 집중되어 있습니다.)
